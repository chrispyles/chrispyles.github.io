Work Samples

As quick sample of the kind of work that goes on in UC Berkeley introductory CS classes, here is one of the homework problems I completed in CS 88: it is a function that computes the hailstone sequence of a number n and returns the number of steps in that sequence. A hailstone sequence goes like this:

While n is not 1:

1. If n is even, divide it by 2.

2. If n is odd, multiply it by 3 and add 1.

3. Repeat until n is 1.

Here is the program I wrote to find the hailstone sequence of a number n (note that the code follows the same sequence of actions as the description above):


def hailstone(n):

print(n)

steps = 1

while n != 1:

if n % 2 == 0:

n = int(n/2)

print(n)

steps += 1

else:

n = int(3*n+1)

print(n)

steps += 1

return steps


That function uses iteration (a while loop) to go through the values of n until it gets to 1. However, you can also define the hailstone function recursively. A recursive function calls itself in its body, and follows the model of mathematical induction (a proof strategy in which a proposition's base case is proven and then an arbitrary step n is assumed to be true and is used to prove that the step n + 1 is true).


def recursive_hailstone(n):

print(n)

if n == 1:

return 1

else:

if n % 2 == 0:

n = int(n/2)

return recursive_hailstone(n) + 1

else:

n = int(3*n+1)

return recursive_hailstone(n) + 1


If you ran hailstone(10) or recursive_hailstone(10), you would get this output:


>>> a = hailstone(10)
10
5
16
8
4
2
1
>>> a
7

>>> a = recursive_hailstone(10)
10
5
16
8
4
2
1
>>> a
7
